## 1. 当前结论

方向 A 的重点不是某一个框架，而是一条完整工程链：**Python/Java 工程基础 → Agent Loop 与 Tool Calling → RAG → 上下文、记忆和工作流 → Evaluation → 部署、可观测性与安全**。

Java 不需要放弃。27 个正式岗位中有 11 个明确提到或接受 Java；最有效的转型策略是补齐 Python 与 LLM 应用工程，同时把 Spring Boot、MySQL、Redis、并发、消息队列、鉴权、日志和生产排障迁移到 Agent 项目中。

### S 级：现在就学，决定能否进入方向 A

1. **Python 工程 + 保留 Java 后端优势**：Python 用于 Agent 服务与生态，Java 用于企业系统集成和稳定后端。
2. **Agent Loop + Tool Calling**：必须理解状态、决策、工具 Schema、参数校验、最大步数、重试和失败退出，不能只会调用框架。
3. **RAG 全链路**：文档解析、切片、Embedding、混合检索、重排、引用、评估和坏例分析。

### A 级：从 Demo 走向生产的分水岭

4. **Prompt 与上下文工程**：结构化输出、上下文裁剪、版本管理、注入防护和成本控制。
5. **记忆与状态工作流**：短期/长期记忆、Checkpoint、恢复、幂等、人工审批和多智能体协作。
6. **Evaluation**：任务完成率、工具选择正确率、检索命中、延迟、Token、失败分类与回归测试。
7. **AI 后端与生产化**：FastAPI、Spring Boot、MySQL、Redis、消息队列、SSE、Docker、Tracing、限流、降级和告警。

### B 级：主线跑通后补齐

8. **MCP 与外部系统集成**：有增长趋势，但当前仅 7/27；先会 Tool Calling，再学习 MCP Server/Client 和权限边界。
9. **安全与治理**：频率低不代表可忽略；在有副作用的工具、企业数据和生产部署中是上线门槛。

### C 级：暂缓

- LoRA、RLHF、DPO、Agentic RL、GPU 调优、分布式训练和推理集群不是当前主线。模型训练相关能力在 9 个岗位出现，但只有 3 个列为职责或必备，且集中在混合/相邻岗位。
- 知识图谱、大数据和多模态按目标岗位再补，不先于 Agent、RAG、评测和生产化。
- 不同时学习多个 Agent 框架。先写无框架 Agent Loop，再选择 LangGraph 做状态化重构。

## 4. Java 后端的迁移策略

| 已有能力 | 在 Agent 项目中的新用法 |
| --- | --- |
| Spring Boot / REST | 业务网关、鉴权、会话、配额、SSE 流式输出 |
| MySQL / Redis | 任务状态、Checkpoint、缓存、幂等键、配额与审计 |
| MQ / 异步任务 | 长任务解耦、重试、降级、事件驱动工具执行 |
| 微服务与并发 | Agent 服务隔离、限流、熔断、成本与延迟优化 |
| 日志与排障 | Trace 串联模型、检索、工具和业务调用，建立失败分类 |
| 测试 | Tool 契约测试、RAG 回归集、Agent 轨迹测试和坏例回放 |

必须补齐的是 Python 类型与异步编程、FastAPI/Pydantic、LLM 结构化输出、RAG、Agent 状态控制和评测方法，而不是重学一遍普通后端。

## 5. 学习路线摘要

- **第 1–4 周**：Python/FastAPI、LLM API、结构化输出和可测试的无框架 Agent Loop。
- **第 5–8 周**：Tool Calling、RAG、引用与 50 条评测集，完成第一个可演示闭环。
- **第 9–12 周**：LangGraph、状态/记忆、Checkpoint、人工审批和自动回归。
- **第 13–16 周**：Spring Boot 集成、MySQL/Redis、SSE、Docker 与可观测性。
- **第 17–20 周**：MCP、安全、权限、Prompt Injection、性能与 Token 成本优化。
- **第 21–24 周**：项目打磨、架构文档、量化结果、面试题和定向投递材料。

完整的每阶段目标、验收标准和项目证据见 `00-dashboard/ROADMAP.md`。
