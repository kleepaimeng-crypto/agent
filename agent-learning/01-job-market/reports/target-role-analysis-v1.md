### 稳定交集

1. **不是会调用一个模型 API 就够了。** 方向 A 的稳定组合是编程基础、RAG、工具调用、Agent 控制循环、Prompt/上下文以及部署可观测性。
2. **社招更强调工具与工程闭环。** Tool Calling 在 10/12 个社招样本出现，工作流在 7/12 个社招样本出现；岗位关心的是业务流程能否稳定运行，而非只会做对话 Demo。
3. **评估已进入主流要求。** 13 个岗位要求 Agent/RAG 评估，12 个将其列为职责或必备；项目应从一开始就有测试问题、指标和坏例。
4. **MCP 是有价值的扩展，但暂不是第一学习入口。** 7/25 个岗位出现 MCP，重要性低于 Tool Calling、RAG、Agent Loop 和后端工程。
5. **模型训练不是当前主线。** 8 个岗位提到模型训练或推理，其中 5 个只作为加分，硬性要求主要集中在混合岗位和两个相邻样本。

## 3. Java 后端转型判断

### 可以直接复用的能力

- 9 个岗位明确接受或提到 Java；J006、J015、J017、J023 给出了直接的 Java/Spring 路径。
- Spring Boot/Spring Cloud、MySQL、Redis、消息队列、REST、Docker、Linux、并发、日志和微服务并没有过时，它们正是 Agent 从 Demo 进入生产的底座。
- 现有 Java 经验最适合证明：接口设计、幂等、重试、权限、数据库建模、缓存、消息队列、服务治理和生产排障。

### 需要补齐的差距

1. Python 工程：类型标注、虚拟环境、Pydantic、asyncio、FastAPI、测试和项目结构。
2. LLM 控制：结构化输出、上下文管理、工具 Schema、最大步数、超时和错误恢复。
3. RAG：文档处理、混合检索、重排、引用、检索评估和坏例分析。
4. 状态化工作流：Checkpoint、恢复、人工审批和幂等执行。
5. Agent 评估与可观测：任务完成率、工具选择正确率、延迟、Token、失败类型和追踪。

### 当前不应优先投入

- 不先以 LoRA、RLHF、并行推理、GPU 调优为主线；它们在本样本中更多是混合/相邻岗位能力。
- 不同时追完所有 Agent 框架。先掌握无框架 Agent Loop，再用一个状态工作流框架重写。
- 不把 Dify/Coze 的拖拽使用当作完整工程能力；需要补插件、接口、部署、评估和错误处理。

## 4. 未来两个月前 10 项学习优先级

| 顺序 | 学习任务 | 对应岗位信号 | 必须形成的项目证据 |
| ---: | --- | --- | --- |
| 1 | Python 工程与 FastAPI | Python 在 22 个岗位的原始技能中出现 | 类型完整的异步 API；含测试、超时、错误处理和日志 |
| 2 | LLM API、结构化输出与上下文 | LLM 基础 15，Prompt/上下文 16 | 记录 Token、耗时、无效响应和上下文裁剪的最小服务 |
| 3 | 无框架 Agent Loop | Agent Loop 17，Tool Calling 18 | 两个以上工具；最大步数；参数校验；失败重试；完整轨迹 |
| 4 | RAG 全链路 | 20/25 岗位涉及 RAG | 文档切分、向量+关键词检索、重排、引用和 30–50 条测试集 |
| 5 | 状态、记忆与工作流 | 状态/记忆 14，工作流 12 | Checkpoint、任务恢复、条件分支和人工审批 |
| 6 | Agent/RAG Evaluation | 13 个岗位出现评估 | 完成率、工具正确率、检索命中率、延迟、Token 和坏例报告 |
| 7 | Java 与 Agent 服务集成 | 9 个岗位接受 Java；4 个直接指向 Java 后端 | Spring Boot 调用 Python Agent 服务；含 SSE、鉴权、数据库和 Redis |
| 8 | 部署与可观测性 | 16 个岗位要求部署/观测 | Docker Compose、Tracing、指标、日志、重试、限流和成本看板 |
| 9 | MCP 与外部系统 | 7 个岗位出现 MCP | 一个 Project Diagnostics MCP Server，暴露文件、Git、端口等安全工具 |
| 10 | 安全、权限与审计 | 频率较低但属于生产门槛 | 工具白名单、危险操作审批、Prompt Injection 测试和审计记录 |

### 两个月建议节奏

- 第 1–2 周：完成 Python/FastAPI、模型 API、结构化输出和最小 Agent Loop。
- 第 3–4 周：加入 RAG、引用和第一版评估数据集。
- 第 5–6 周：用状态工作流重构，加入恢复、审批、失败测试。
- 第 7 周：接入 Spring Boot、MySQL/Redis、SSE 与鉴权。
- 第 8 周：完成 Docker Compose、Tracing、指标、成本统计和项目说明。
